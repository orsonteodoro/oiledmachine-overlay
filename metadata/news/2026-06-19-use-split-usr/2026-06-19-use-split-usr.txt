It is decided to reverse the decision to ban merge-usr.  systemd will allowed but
unchecked in on this overlay.  We will require systemd be live.

The reason why this overlay decided to make systemd a soft ban is for the
following reasons:

- It is using a memory unsafe language.
- It increases the attack surface by preventing read only mount, nosuid, nodev
  mount flags.
- Unpatched security commit backports in tagged releases by distro maintainers.
- It is a performance first and not a security first project.  If it was security
  first, it would have already been re-written in Rust.
- To reduce the init vulnerabilities.
- To reduce maintenance burden.
- The OpenRC scripts are more tested than the systemd.
- We want to halt the monolithic distro trend were users are forced and locked-in
  to use the popular distro components.
- We like to utilize the split-usr advantage of the mount flags.

While merge-usr will be the only choice, users should be aware about the flaws in
order to protect their systems.

MAC (Mandatory Access Control) rules must be written more comprehensively and
reviewed for consistency for /bin, /sbin, /lib, /lib64, /usr/bin, /usr/sbin,
/usr/lib, and /usr/lib64.  A MAC profile maintainer may assume merge-usr layout
and write fewer rules so the protection may be inadequate for split-usr.

If you use a split-usr, nothing needs to be done except for staying away from
systemd and merge-usr.

It split-usr be supported as long as the distro supports it and projects support
it, but it looks like they are likely to drop it in the future if the devs are
all compromised by Fedoraization and that is the trend.


Only do this if you are a systemd user:

For live systemd, you need to hard unmask it.  The commit history for systemd
reveals a vulnerability.  This is why live systemd is required on this overlay
for security integrity.  To use the live systemd, the following need to be
edited:

Contents /etc/portage/package.accept_keywords:
=sys-apps/systemd-9999:0/2::oiledmachine-overlay ~amd64 **

Contents /etc/portage/package.mask:
sys-apps/systemd::gentoo

Contents /etc/portage/package.unmask:
=sys-apps/systemd-9999:0/2::oiledmachine-overlay

The distro is still stuck in v261 RC3 but it is already final v261.  The live
ebuild with the security fix(es) (e.g. 17260a9) has the commits for v261
systemd final.

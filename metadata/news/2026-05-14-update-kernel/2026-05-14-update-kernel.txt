The Linux kernel had several zero day vulnerabilities:

CVE-2026-46300 - Fragnesia: privilege escalation, attack chain primitive
CVE-2026-43284, CVE-2026-43500 - Dirty Frag:  privilege escalation, attack chain primitive
CVE-2026-31431 - Copy Fail:  priviliege escalation, container escape, attack chain primitive

CVE-2026-43284 - CVSS:3.1/AV:L/AC:H/PR:L/UI:N/S:C/C:H/I:H/A:H (denial of service, data tampering, information disclosure)
CVE-2026-31431 - CVSS:3.1/AV:L/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:H (denial of service, data tampering, information disclosure)
CVE-2026-46300 - String vector TBA

The AV:L (locally ran) is deceptive, it can be chained together with another
exploited vulnerability with remote code execution for full system compromise.
The chances increase if you haven't updated your kernel for months or years or
through unpatched red zone apps.

IMPORTANT:  Always update the initramfs with updated kernel modules to avoid
getting a black bootup screen and the boot disk is mounted to receive the
updates.

Remember to keep backup kernels.  Do not delete old kernels until you verify
that the kernel works.

All solutions assume that after the the kernel as been installed, a new updated 
initramfs generated for the kernel flavor, boot config updated; a reset is done.
After reset the disabled .config solution is done; but for the external module
techniques they should either prevent the loading of the affected code or
require explicitly unloading every reboot until the official fix is applied.

====================
Remediation provider
====================

Fragnesia - Genpatches
Dirty Frag - Linux kernel update
Copy Fail - Linux kernel update

===========
Remediating
===========

Download either ot-sources or gentoo-sources.

Verify the following kernel versions to apply full mitigation for all named 3
vulnerabilities.

Get the patched version(s)

     >= 7.0.7
     >= 6.18.30
     >= 6.12.88
     >= 6.6.138
     >= 6.1.172
     >= 5.15.206
     >= 5.10.255

Genpatches will apply the Fragnesia mitigation.
The Linux kernel team already applies Dirty Frag and Copy Fail mitigations.

This solution is the most preferred because it also triages several low level
critical-high vulnerabilities that do not get the same media attention.

The distro servers are slow so be patient when receiving patchsets.

Using gentoo-kernel or gentoo-kernel-bin with KFENCE are okay as a balanced
security kernel or even as a backup recovery kernel, but it is perferred that
gentoo-sources be used and configured with KASAN with panic_on_warn=1 with a
higher security score for security-critical use cases.

=======================================
The mitigation status after remediation
=======================================

After applying the patches, updating initramfs, and rebooting, the status of
each vulnerability is as follows:

Fragnesia - Fully mitigated
Dirty Frag - Fully mitigated
Copy Fail - Fully mitigated

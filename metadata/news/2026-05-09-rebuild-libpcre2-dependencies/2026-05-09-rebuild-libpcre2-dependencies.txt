We have set --disable-symvers on dev-libs/libpcre2 to avoid widespread
build failures with packages like glib, qtbase, webkit-gtk, etc.

Action required:
After syncing this overlay and re-emerging libpcre2, you should rebuild
all packages that depend on it:

    emerge --ask -o1vuDN $(equery depends -F '$cp' dev-libs/libpcre2 | sort | uniq)  # Install missing dependencies
    emerge --ask -1vO $(equery depends -F '$cp' dev-libs/libpcre2 | sort | uniq)     # Force rebuilding the direct dependencies linking to all libpcre2 libraries

or simply:

    emerge -1v @preserved-rebuild

This change eliminates the "no version information available" warnings
and undefined reference errors caused by PCRE2 10.47 symbol versioning.

This rebuild request is made to avoid hidden build inconsistencies or
random load time issues.
